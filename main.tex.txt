\section{Introduction}

\noindent Home networks are full of sensitive data --- yet most are completely unmonitored.
This project turns a \textbf{Raspberry Pi 5} into a self-hosted security gateway,
combining network-wide ad blocking, encrypted remote access, and real-time device
monitoring --- with no subscriptions and no third parties involved. Rather than
building from scratch, it integrates two proven open-source tools:
\textbf{Pi-hole} and \textbf{WireGuard}, unified through a custom backend and dashboard.

\medskip
\noindent\textbf{Traffic flow:}\\
\texttt{Device} $\rightarrow$ \textbf{Pi-hole} (DNS filtering \& ad blocking)
$\rightarrow$ \textbf{WireGuard} (encrypted VPN tunnel for remote access)
$\rightarrow$ \textbf{FastAPI + Dashboard} (management, alerts \& monitoring)

\medskip
\noindent\textbf{Key properties:}
\begin{itemize}
  \item[-] 3 user roles: Admin (full control), User (VPN + stats), Guest (read-only)
  \item[-] Client-server architecture with a relational database (5 tables)
\end{itemize}

\subsection{Database Diagram}

\noindent The database consists of five tables. Relationships:
\begin{itemize}
  \item[-] \texttt{users} $\rightarrow$ \texttt{vpn\_connections}: \textbf{1 -- n}
  \item[-] \texttt{users} $\rightarrow$ \texttt{devices}: \textbf{1 -- n}
  \item[-] \texttt{devices} $\rightarrow$ \texttt{alerts}: \textbf{1 -- n}
  \item[-] \texttt{devices} $\rightarrow$ \texttt{adblock\_logs}: \textbf{1 -- n} (Pi-hole sees devices, not users)
\end{itemize}
