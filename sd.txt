# Project Title: Pi-Based Network Security & Privacy Gateway

## Project Overview
This project turns a **Raspberry Pi 5 (4GB RAM)** into a **home network security and privacy gateway**.  
The system combines:

- **VPN server** (WireGuard/OpenVPN) for secure remote access  
- **Pi-hole** for network-wide ad-blocking and DNS filtering  
- **IoT & LAN device monitoring** with alerts for unusual activity  
- **Web dashboard** to manage users, devices, and network statistics  

The project satisfies Software Design course requirements: it has a **client-server architecture**, **3 user roles**, a **relational database**, and is designed using **OOP principles**.

---

## User Roles & Permissions

| Role      | Permissions |
|-----------|-------------|
| **Admin** | Full control: configure VPN and Pi-hole, add/remove devices, view all logs, manage alerts and security rules |
| **User**  | Connect to VPN, view personal stats, access Pi-hole basic stats |
| **Guest** | Read-only dashboard access; view network activity summary |

---

## Core Database Tables

| Table Name       | Purpose |
|-----------------|---------|
| `users`         | Stores user credentials, roles, and VPN access information |
| `devices`       | Tracks LAN/IoT devices: IP, MAC, hostname, last seen |
| `vpn_connections` | Logs VPN usage per user (start/end time, duration, status) |
| `adblock_logs`  | Stores Pi-hole blocked requests and query statistics |
| `alerts`        | Stores alerts triggered by unusual network activity or security rules |

> Optional: `roles` table for mapping role-based permissions.

---

## Technical Architecture

**Backend:** Python + FastAPI  
- Handles API endpoints for user/device management, alerts, VPN logs, and Pi-hole data  

**Frontend:** HTML/CSS/JS or React/Vue  
- Displays live dashboards, charts, and tables for devices, VPN usage, and Pi-hole stats  

**Database:** SQLite (lightweight) or PostgreSQL (robust relational DB)  

**System Integration:**  
- Pi-hole runs in a Docker container for isolation  
- VPN server runs via WireGuard or OpenVPN  
- Backend scripts periodically fetch Pi-hole and VPN logs  
- IoT/network devices are monitored internally on the LAN  

---

## Use-Case Highlights

- Admin: configure VPN, manage Pi-hole, add/remove devices, view all alerts  
- User: connect to VPN, view dashboard stats, see Pi-hole queries  
- Guest: view summary dashboard, receive notifications (read-only)  

---

## Security Considerations

- VPN connections use **strong encryption** and keys/passwords  
- Pi-hole admin panel is restricted to **LAN or VPN access only**  
- Web dashboard uses **authentication** and **role-based access control**  
- Firewall and OS hardening on Raspberry Pi  
- Sensitive data stored securely (hashed passwords, minimal logging of personal traffic)  

---

## OOP Design Concepts

- **User class**: manages roles, authentication, and VPN connections  
- **Device class**: tracks device information and network activity  
- **VPNConnection class**: manages start/stop logs per user  
- **Alert class**: generates and stores security notifications  
- **AdblockLog class**: stores Pi-hole queries and statistics
